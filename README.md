# SQL Assignments: Business Data Analysis & Database Management

This repository contains a collection of SQL assignments focused on data analysis, database schema design, and complex query operations. These scripts demonstrate proficiency in extracting business insights and managing relational databases.

---

## 📂 Project Overview
The project is divided into three main modules based on real-world business scenarios:
1.  **Sales Data Analysis:** Calculating revenue, profit, and performance trends.
2.  **Product Management:** Creating structured catalogs and handling schema updates (DDL).
3.  **Employee & Department Relationships:** Implementing various JOIN types to link organizational data.

---

## 🚀 Key SQL Concepts Covered

### 1. Sales Performance Analysis
In this section, I analyzed a `sales_data` table to answer critical business questions:
* **Total Revenue & Profitability:** Calculating the overall financial health of the company.
* **Region-wise Performance:** Using `GROUP BY` and `ORDER BY` to identify high-performing and weak regions.
* **Category Metrics:** (Planned) Calculating profit margins per product category.

### 2. Database Schema & Data Entry
Practical implementation of Data Definition Language (DDL) and Data Manipulation Language (DML):
* **Table Creation:** Built `Products`, `Departments`, and `Employees` tables with primary keys.
* **Schema Correction:** Used `ALTER TABLE` to rename columns and fix structural errors.
* **Data Insertion:** Populated tables with sample business records.

### 3. Advanced Data Retrieval (Joins)
Demonstrated how to combine data from multiple tables using different join types:
* **Inner Join:** To list employees only if they belong to a specific department.
* **Left Join:** To show all employees, including those not yet assigned to a department.
* **Right Join:** To list all departments, even if they currently have no employees.
